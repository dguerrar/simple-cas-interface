CAS

new CAS(parameters)

Instances of CAS provide convenient access to protocol URLs like '/login' and '/logout', as well as ticket verification.

Parameters:
Name Type Description
parameters CasParameters An instance of CasParameters. Required.
Properties:
Name Type Description
serverUrl string The base URL for the remote CAS server to which all API methods will be appended.
serviceUrl string The local endpoint that the remote CAS server will interact with, e.g. where clients will be redirected after login.
protocolVersion string The version of the CAS protocol that will be used when communicating with the remote CAS server.
serviceValidateUri string The endpoint that will be used to validate service tickets.
logoutUrl string The full URL, with query parameters, to use when sending clients to the remote CAS server for authentication.

validateServiceTicket(ticket) → {Promise}

After receiving a service ticket from the remote CAS server, use this method to verify its authenticity.

Parameters:
Name Type Description
ticket string The service ticket string as returned from the remote CAS server.
Source:
  • cas.js, line 114
Returns:

A Promise that resolves if the ticket is a valid service ticket. The parameter passed to the resolve function will be an object the represents the authenticationSuccess elements of the returned XML for protocols 2.0 and 3.0. For protocol 1.0 it will merely be true.

If the ticket it is invalid, or some other issue occurred, the promise will be rejected with an instance of Error that indicates why.
